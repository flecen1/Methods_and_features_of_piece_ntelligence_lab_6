## Запуск з командного рядка

```powershell
python main.py --demo-toy --json
python main.py --variant 1 --noise 0.4 --trials 50 --seed 7 --json
python main.py --variant 1 --foreign V --json
python main.py --variant 1 --plot-triptych B --noise 0.55 --plot-file figures\variant1_b.png --plot-energy --no-show
```

Параметри:

- `--demo-toy` — демонстрація на **2 нейронах** і **4 біполярних образах** (кути квадрата); перевірка стабільності еталонів.
- `--variant` — номер **1…20** згідно з **таблицею варіантів** (див. нижче; у коді — `variants.py`).
- `--noise` — коефіцієнт **σ** гаусового шуму у виразі `L + σ·N(0,1)` перед переходом до біполярного старту.
- `--trials` — скільки випадкових спотворень зробити **для кожної** з трьох літер варіанту.
- `--seed` — відтворюваність випадкових чисел (якщо не вказано, серії будуть інші — це нормально для статистики).
- `--foreign` — одна велика літера **A…Z**, якої **немає** серед трьох літер вашого варіанту; подається **без** додавання шуму.
- `--alphabet` — шлях до альтернативного `Alphabet.csv`, якщо файл не в теці проєкту.
- `--json` — вивід результату у JSON (зручно копіювати у звіт).
- `--plot-triptych` — літера з **вашої** трійки для трьох панелей: еталон / спотворений вхід / після мережі.
- `--plot-file` — зберегти PNG; `--plot-energy` — додатково графік енергії; `--no-show` — не відкривати вікно matplotlib.

Перед збереженням рисунків створіть теку, наприклад:


## Відповіді на контрольні питання (коротко)

1. **Які задачі розв’язує мережа Хопфілда?** Асоціативна пам’ять, відновлення спотворених дискретних образів, рух стану до локальних мінімумів енергії.

2. **Умови стійкості (класична модель)?** Симетричні ваги, нуль на діагоналі, асинхронне оновлення — енергія не зростає, траєкторія сходиться до стійкого стану.

3. **Недоліки?** Обмежена ємність пам’яті, хибні атрактори, чутливість до схожих образів і до великого шуму.

4. **Енергетична функція?** Оцінює «якість» поточного стану; при асинхронних оновленнях енергія зменшується.

5. **Коли зупиняється програма?** Після повного проходу по нейронах стан не змінився, або досягнуто ліміту ітерацій у коді.

## Примітка щодо `--demo-toy`

Для **двох нейронів** і **чотирьох** біполярних «кутів квадрата» класичне правило Хебба з **обнуленням діагоналі** дає **нульову** матрицю ваг `W`, тому динаміка стає тривіальною. У функції `demo_toy` діагональ **тимчасово не** обнулюється (`zero_diagonal=False`), щоб наочно показати ненульові ваги й стабільність еталонів. Для **літер** (35 нейронів) у проєкті завжди використовується **стандартне** обнулення діагоналі після обчислення `W`. У звіті варто коротко згадати цю відмінність, якщо порівнюєте з «класичною» формулою на папері.

## Що включити у звіт

1. Результат `python main.py --demo-toy --json` (текст або скрін).
2. Таблицю або графік **точності** (`accuracy`) від **σ** для **кожної** з трьох літер вашого варіанту (кілька рівнів σ, однакові `trials`, за можливості один `seed` для порівняння).
3. Результат `--foreign` для літери **поза** вашою трійкою + короткий висновок (до якого еталону «тягне» мережа).
4. Хоча б **один** PNG (triptych або скрін GUI); за бажанням — графік енергії.
